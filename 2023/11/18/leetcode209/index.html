题目描述

209.长度最小的子数组

给定一个含有 n 个正整数的数组和一个正整数 target 。

找出该数组中满足其总和大于等于 target 的长度最小的 连续子数组 [numsl, numsl+1, ..., numsr-1, numsr] ，并返回其长度**。**如果不存在符合条件的子数组，返回 0 。

提示：

  • 1 <= target <= 109
  • 1 <= nums.length <= 105
  • 1 <= nums[i] <= 105

解题方式

暴力

第一眼看到此题，你可能会尝试暴力，则枚举iii和jjj，然后判断子数组里的元素是否>=target>=target>=target

时间复杂度:O(n2)O(n^2)O(n2)

空间复杂度:O(1)O(1)O(1)

但是很遗憾，暴力超时了，我们看回提示中的1<=nums.length<=1051<=nums.length<=10^51<=nums.length<=105，就是数组范围最坏情况是10510^5105个元素，这样双层循环肯定会超时

因此我们需要把O(n2)O(n^2)O(n2)的时间复杂度优化到O(n)O(n)O(n)

滑动窗口

  • Q:什么情况下用滑动窗口
  • A:像这样的子数组问题，或者连续区间问题，都可以用滑动窗口

滑动窗口本质就是定义左右边界，然后枚举右边界，维护左边界，并在满足条件下更新答案

首先定义iii和jjj,iii为左边界,jjj为右边界，再定义一个无穷大的答案ansansans一般为10910^9109,以及一个ttt ttt代表子numsnumsnums里的元素和,然后开始思考如何维护左边界，如何更新答案

先回答第一个问题

  • 当t>=targett>=targett>=target时，我们去缩小左边界，查找有没有更小长度的子数组，并更新ansansans

第二个问题

  • 当iii在缩小的时候，判断t>=targett>=targett>=target,更新当前答案为j−i+1j-i+1j−i+1，注：当出现只有一个元素的时候 要加上自己 所以要+1

明白这些问题后，我们就开始写代码了

首先定义iii,jjj,ansansans,ttt

写一个循环枚举右边界jjj

开始维护子数组的元素，并加上维护iii

这样我们就遍历了一次数组解决了问题 可以通过本题了
